  2. If the user is already logged in, they are automatically redirected to the /dashboard.
  3. If not, the user fills out the login form with their credentials and submits it.
  4. The system checks for too many failed login attempts from the same IP address.
  5. If the credentials are valid, the user session starts and they are redirected to the dashboard.
  6. If the credentials are incorrect, an error message is shown: Invalid email/username or password.
